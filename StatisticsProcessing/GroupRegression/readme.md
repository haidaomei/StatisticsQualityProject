交互项回归和检验.docx

现在来讲一下我们的交互项检验
原文是这么讲的: 按粮食主产区/非主产区、气候带、地形梯度等维度进行分组回归、交互项检验与分位数回归 其实产生交互项本身就是一个回归 所以这里这句话我感觉应该分解成1.分组 2.交互项回归和检验 3.分位数回归才对
重申一下我们的双向固定效应面板模型 它的形式是
Y_单产=黑龙江省固定自变量成分*bool_黑龙江+北京固定自变量成分*bool_北京+广西固定自变量成分*bool_广西+2023固定自变量成分*bool_2023+2022固定自变量成分*bool_2022+2021固定自变量成分*bool_2021+因年省都改变的自变量成分(这部分就是f(灾害)、g(机械化)、h(土地面积)等等)
其实本身双向固定效应面板模型也是一种回归 它和我们高中学的用最小二乘法进行的线性回归一样 然后我之前提到了我们的自变量要有f(灾害)、g(农机动力)、h(化肥施用量)、u(有效灌溉面积)、v(农药使用量)、α(农用薄膜量)、β(除涝面积)
那什么是交互项检验?其实很简单 我们想分析灾害和农机动力二者同时产生的影响 我们就创造一个单项式灾害*农机动力 然后把它丢到公式里面 在因年省都改变的自变量成分里添加就行了
比如添加了农机动力和化肥使用量交互的项 我们的因年省都改变的自变量成分会变成f(灾害)+g(机械化)+h(土地面积)+a(农机动力*化肥施用量) 注意我们的因变量仍然是Y单产
算法实现的具体原理我不知道 不过我们之前不是有分析过热力图吗 我们说对于两个自变量 我们可以把它抽象成三个因素 这三个因素排列组合来呈现在两个自变量上 所以线性回归的时候对有相关性的变量 会重复统计底层因素 造成回归出现偏差
但是这里交互项正是要放大这个因素 我们现在有因素123去映射到农机动力和化肥施用量 比如农机动力对应因素12 化肥施用量对应因素23 相乘之后这个2对因变量的影响就大大增加了 我们同样可以从交互项的系数看出交互项对因变量的影响
这个系数越大 说明交互项二者重合程度就越高 说明这二者同时发生对单产的影响是upupup的
再比如假设我们有两种灾害自变量 我们做一个旱灾*洪涝的交互 结果发现这个项的系数是非常负的值 说明这二者合起来对单产的影响是极其降低 比两个灾害普通叠加起来造成的灾害还要大 这就是交互项的意义了
那我们现在有7个自变量 要做C_7^2=21个交互项吗?
不用 也不应该做 因为有些项交互起来是无意义的 比如化肥*薄膜 我问这二者交互对单产有什么意义 有人可能说是加强单产 但是这和我们的灾害分析没关系啊 我们看的是灾害对单产的影响 又不是看增产
我在这里提出几个值得的交互项:灾害*有效灌溉面积、灾害*除涝面积、灾害*农机动力
当我们得出了这几个交互项的系数 我们就可以说明灌溉和除涝面积是不是真的能减灾 机械化能不能提升应急响应能力
所以这里也奠定了我们交互项的基调 就是要跟灾害交互 还有其他的比如灾害*化肥或者薄膜就没必要 个人感觉它们不是很宏观 但是做起来其实也没什么问题 总之要符合我们一定要和灾害项交互的基调
做交互其实也很简单 我们直接生成一列灾害*有效灌溉面积当作因年省都改变的自变量丢到双向固定效应面板模型去回归就行了:]
