其实用这玩意去搞出一个可信度p就是用shap和小变换 行为是假如河南一年shap总值是n 那我们直接把logistic分母的那个n个变量线性组合的指数换成shap总和即可 然后我们对每个点绘制出它的置信度p

logistic模型这里略过(因为我学过了= =)

我们先看那个2*2列联表 我们有误报的情况(本来丰收但是搞成灾害了) 也有漏报的情况(本来灾害报成丰收了) 其他就是不误报(该灾害灾害该丰收丰收) 所以有错误和正确两个

然后我们对那些p值的元组排个序 p最大放在最上面 p最小放在最下面

此时我们开始从1到0移动p 我们看在每一个从1到0的p下正确率是多少 错误率是多少 我们以丰收被误报成灾难的概率为x 以灾难被正确汇报的概率为y绘制出roc图像(注意这里的概率不是p 而是我们以我们设定的p得出一个2*2列联表里列联表那些数的比例) 这就是绘制方法

如果有一段垂直向上的线说明了什么?说明我们的p仅仅是动了一点点 就能捕捉到一堆灾难(正确率不断上升 错误率一点没提)

如果有一条水平的线说明我们的p移动了很多也没能捕捉到灾难(错误率不断上升正确率一点没提)

(其实对于这种情况 我更喜欢画出p和正确率+p和错误率 那个比较直观 但是捕捉不到正确率和错误率的博弈 所以才会有这个ROC曲线)

对于一条y=1的线 我们可以看到我们在移动p的时候它的错误率一直在提高 正确率却没变 说明这个模型的精度就到头了 我们在拐到这条y=1之前的那个点 或者在垂直上升结束之后的那个点对应的p就是最佳p 然后垂直上升这就说明这条线会离图中心很远 ks检验正是以这种方式求出一个好的p
