我希望用一些流程图总结一下我们的这个数据分析流

首先就是抓数据了 抓完数据先手动删了一些大表阶段就和粮食生产没关联的 然后再根据关键词定位删掉和粮食生产没有关系的指标(这里误杀了农药) 之后识别系统性误差 从全国全省剔除系统性误差的指标 之后对剩下的指标线性插值线性外推 3sigma抹峰再插值 这里注意如果sigma在统计学上为0那么是保留的 而不是抹除全部 之后对一些为负的指标都设0 一些只能整数的指标化为整数 最后加上农药数据

上面除了抓数据就是数据预处理的部分 分为三大部 第一步是手动删粮食生产没关联的 第二步是根据关键词定位删指标 第三步是线性插值外推抹峰再插值 小的部分就是sigma为0的处理 负的处理 特殊单位非整的处理 农药数据的加入

之后来到我们的数据分析 我们首先是对所有预处理完之后的数据进行pearson和spearman相关性分析并绘制热力图 之后再次手动筛选那些关键的能表征我们产量目标的数据 并确保这些数据不同维度 比如单产和产值我们选单产 机械化也有很多备选 最终选择什么来着(纵观全局我们是手动筛两次 关键词筛一次)

热力图绘制完后我们进行的就是固定效应模型的建立 这应该没什么可以取舍的 对我们的那些自变量做就对了 不过我们的表头不是虚拟变量 而是化为01的bool值 因为数学公式识别不了虚拟变量

做完固定效应模型的分析后是按主产和非主产 气候区 三大地形做一个分组的交互项的固定效应模型回归 之后全国一盘棋做分位数回归

接下来就是机器学习回归 我们用了xgboost和随机森林还有滞后特征 用了shap值和部分依赖图

最后的内生性克服用了双重差分和合成控制 这两个分析方法都是针对特定事件的 我们选取2023河南的烂场雨作为例子

接下来就是滚动预测 我们用去年数据预测今年 用了xgboost模型 之后就是混淆矩阵 roc曲线 ks检验这些我都放在项目里

最后的最后 webgis可视化 数据待填上去

总览我们的各个技术流程:爬->手筛->机筛->系统性误差筛->线性插值和外推->数据合理性转化->补充数据->pearson和spearman相关性分析及热力图->手筛->固定效应模型->分组->交互项固定效应模型->分位数回归->xgboost和随机森林机器学习->shap和部分依赖图->双重差分和合成控制->xgboost滚动预测->混淆矩阵、roc曲线、k-s检验->web可视化
